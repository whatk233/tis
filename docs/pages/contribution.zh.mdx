# 贡献

TIS 的调整规则非常需要社区的支持，因为单凭个人是没有那么多精力来管理那么多规则

## 贡献者

[![](https://contrib.rocks/image?repo=whatk233/tis)](https://github.com/whatk233/tis/graphs/contributors)

## 添加规则

你可以给 TIS 的调整规则添加一砖一瓦

### 必备条件

- Github 账号
- Git 基本使用

### 贡献方式

1、Fork 项目，创建一个分支

2、根据“[规则文件语法](/zh/advanced/ruleSyntax)”新建调整规则，并且添加到 `/tweaks` 文件夹中

3、`git push` 后前往 [TIS 项目 Github](https://github.com/whatk233/tis) 中创建一个 pr (pull request) ，接下来等待合并即可！😊 如有疑问项目成员会在 pr 中与你进行讨论。

<Callout type="info">
  存在困难？没关系，你可以将你的调整思路分享到 "[Issues](https://github.com/whatk233/tis/issues)" 中，开发者会手动进行合并 😊
</Callout>

## 调整建议

该规则存在可调整的空间？有更好的设置方法？或者在系统更新后该规则失效了？

你可以 Fork 项目，创建一个分支，修改对应规则后创建一个 pr (pull request)，接下来等待合并即可！😊 如有疑问开发者或志愿者会在 pr 中与你进行讨论。

<Callout type="info">
  存在困难？没关系，你可以将你的调整建议分享到 "[Issues](https://github.com/whatk233/tis/issues)" 中，开发者会手动进行合并 😊
</Callout>

## 问题反馈

规则失效了？规则存在副作用？规则导致部分系统功能异常？

创建一个 "[Issues](https://github.com/whatk233/tis/issues)" 让我们知道该问题，开发者或志愿者会与你进行讨论
